# Reference Documents

When working on architecture, contracts, or UI patterns, consult these documents:

- **`docs/fullstack-foundation-conventions-steps-1-to-5.md`** — HTTP client layer, API contracts, session management, user feedback, and async states
- **`docs/design-system-foundation-checklist.md`** — UI foundations, component primitives, system components, HTTP handling, API contracts, auth/session, and async state checklist

## When to reference

- Planning a new feature that spans API and frontend
- Changing the HTTP client or API module patterns
- Adding or modifying system components (layout, modals, toasts)
- Implementing new async states (loading, empty, error)
- Modifying authentication or session flows
- Adding new UI primitives or changing design tokens
