# Documentation Conventions

The `docs` directory stores project conventions, checklists, and architecture notes that support implementation work.

## Writing guidance

- Keep documents concise, ac
- Update existing docs before creating duplicate documents on the same topic
- Use checklists and clear sections when documenting repeatable engineering practices

## Content expectations

- Reflect actual project paths and scripts
- Call out decisions, tradeoffs, and validation steps when they matter
- Keep frontend, backend, and realtime guidance consistent with each other
